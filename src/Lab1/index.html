Lab. 1 — Basics

Descriptions and definitions

  • HTML (Hypertext Markup Language) — is a standardized markup language that is transferred via HTTP (Hypertext Transfer Protocol) and displayed on the user side
  • CSS (Cascading Style Sheets) — is a styling language that helps in formatting the HTML elements on the output
  • JavaScript — is a scripting language that is capable of performing various client-side tasks (as well as server side ones) affecting the HTML/CSS, and is supported by all modern browsers nowadays
  • HTTP — Hypertext Transfer Protocol is a protocol which in a certain manner works as a compiler for HTML text
  • HTML5 — latest HTML version which mainly intends on restricting extra add-ins and plug-ins and gathering together three pillars of web — JS, CSS and HTML that are aimed to be supported globally
  • Dynamic HTML (DHTML) — is a set of techniques that make web page vary without reloading. This has nothing to do with backend logic, but is achieved with help of Document Object Model architecture, where JS can asynchronously affect HTML and CSS

Coding conventions

"Coding conventions" is something programmer must turn into personal routine. Sure I had my conventional habits, but what I would like to point out is:

  • 1. I write new curly bracket on new line, not on the same e.g. 
    
                for (var i=0; i<10; i++)
                {
                  ...
                }
    Although, I felt used to it, I am working on giving a popular way its try. So from now on, I'll be trying to use the common indentation 
    
                for (var i=0; i<10; i++) {
                  ...
                }
  • 2. I realized using TABs for indentation is not the best practice therefore double-space is to be used by me!
  • 3. I am more comfortable with underscore separated variable names such as number_one instead of camel case numberOne. But that is still something I am trying to figure out for myself. I realized it depends also on the programming language itself. For example, In JavaScript people mostly use camelCase. So let's see which one will fit me better
That's it. I think I am quite comfortable with how I create and take care of my code.
